# Что изменено

## 1) Добавлен третий адаптер в гибридную ветку
- В проект перенесён режим `psych` из второго архива.
- На уровне Django и фронтенда теперь доступны три адаптера: `business`, `legal`, `psych`.
- Для нового адаптера создан каталог `adapters/psych_adapter/`.

## 2) Улучшено гибридное использование
- Вместо фиксированного режима `hybrid` пользователь теперь может включать один или несколько адаптеров одновременно.
- Выбор идёт прямо из интерфейса с помощью toggle-кнопок.
- Для каждого чата сохраняется собственный список активных адаптеров.

## 3) Что нужно сделать на Colab-сервере
В `colab_server.ipynb` уже подготовлена логика под 3 адаптера и динамическую сборку гибридов.
Но фактические веса третьего адаптера отсутствовали в обоих архивах. В исходниках был только пустой каталог `psych_adapter/`.

Поэтому перед запуском нужно положить реальные файлы LoRA-адаптера сюда:
`/content/drive/MyDrive/adapters/psych_adapter`

Обычно там должны быть файлы наподобие:
- `adapter_config.json`
- `adapter_model.safetensors` или `adapter_model.bin`

## 4) Ограничение совместимости
Если третий адаптер обучался не на той же базовой модели или с несовместимыми LoRA-параметрами, смешивание нескольких адаптеров может завершиться ошибкой.
В ноутбуке добавлен безопасный fallback: сначала пробуется одновременная активация, затем weighted-combo, и только если обе схемы несовместимы — сервер вернёт понятную ошибку.
